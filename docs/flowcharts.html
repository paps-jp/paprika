アーキテクチャ・フロー

分散ブラウザフリート Paprika の主要動線(Hub / Worker / Session / Codegen ループ / アセット / マルチハブ / AI 連携)を、系統別の Mermaid 図でまとめた索引ページ。

概要
  • 3 つの永続層 + コーディネーション層: ジョブと設定は MariaDB、アセットは MinIO、ワーカー登録・セッション所有・設定 invalidate は Redis 経由。Hub のローカルディスクは bounded cache(=SoT ではない)。
  • クライアントは round-robin / ワーカー WS は sticky: nginx が複数 Hub を均す一方、worker_id ハッシュで WS だけは一定 Hub に貼り付け、ハブ間の状態差をシンプルに保つ。
  • 動線は 7 系統に分割。各章は独立してジャンプ可能。コードファイルへの参照(file_path:line)を埋めてあるので、図と実装を対応付けて読める。

各章への入口

01

Job Lifecycle

POST /jobs → 直接配信 / queue→再投入 → worker 実行 → WorkerJobComplete → 課題分類 → 永続化。fetchcodegen

 02

Session Lifecycle

OPEN / noVNC RFB keepalive / cross-hub 転送 / 4 経路 CLOSE。session

 03

Worker Fleet

WS 接続 → IP 派生 worker_id → 登録 / version gate + 順次セルフアップデート / heartbeat / ghost 検知。

 04

Codegen Loop

plan → for n: generate → execute → 判定スタック(objective / LLM judge / reasoning) → retry / 終端。codegen

 05

Asset / Storage

worker upload → hub mirror + sidecar → MinIO SoT / 180s evict / DELETE 連動削除。

 06

Multi-Hub

nginx 振り分け / hub_id IP 派生 / 設定 invalidate / session 転送ミドルウェア / P2 再起動復元 / SoT デプロイ。

 07

AI / 課題判定

fetch 完了 → 課題分類(占有率ベース) → escalation 決定 → perception → distiller → barrier→recipe。reviewcodegen
