Worker Fleet

数十台のワーカーが WebSocket で Hub に接続。worker_id は LAN IP から決定論的に派生(w50150)。 version mismatch は Hub PUSH の HubExpectedVersion をきっかけに 順次セルフアップデート。 ghost(プロキシ WS 経由のゾンビ)は両側から検知。

概要
  • worker_id は LAN IP から派生: 10.10.50.150 → w50150。self-update / 再起動 / id-file 喪失を跨いで安定。
  • version gate は heartbeat 経由でゼロダウンタイム: Hub が HubExpectedVersion を毎 heartbeat で push。worker は 順次ドレイン → exit(42) で自己更新。Hub の再起動は不要。
  • ghost 検知は両側に必要: 送信側からは ESTABLISHED に見える failure mode を、Hub 側回収プロセス + worker 側 inbound 沈黙 watchdog の両方で潰す。
