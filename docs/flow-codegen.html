Codegen Loop

mode=codegen-loop のジョブが辿る AI ループ。planner → for n=1..max: generate → execute → 判定スタック(objective pregate → LLM judge → reasoning judge)→ retry context 構築 → 終端。 判定の本丸は server/hub/iterative_codegen.py。

概要
  • plan + execute パターン: 別 LLM が goal を 3–7 step に分解 + テスト可能な success criterion を発行。 以降の試行はその plan に従って 1 スクリプトを書く。
  • 判定は三段: (1) objective pregate(動画ファイル数など物的証拠) → (2) blind LLM judge(screenshot + asset count のみで判定) → (3) reasoning judge(perception → 推論モデル)。 objective が決まれば LLM 判定を skip。
  • retry context は失敗種別ごとに別アドバイス: timeout / zero-progress / under-target / 通常エラーで異なる構造化テキストを次回 prompt に折込む。

6. AI escalation との接続

fetch ジョブが失敗 / completed-but-empty のときに「同じ goal を Codegen ループで挑戦し直す」連携。JobOptions.escalated_from / escalated_to で系譜を辿れる。 発火条件・cooldown・gating は AI / 課題判定 参照。
