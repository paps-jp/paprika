AI Escalation & 課題判定

fetch 完了の中でも 「壁に塞がれて結果が信用できない」 を JobStatus.review に分離(課題)。 AI エスカレーションは failed / review を Codegen ループに再投入。 perception → reasoning distiller でホスト知識を蓄積し、barrier → fetch_recipe で自動学習。

概要
  • 「fetch は完了したが信用できない」を分離: 全画面オーバーレイ(login / age-gate / consent / paywall) で本文が見えていないケースを review(課題)に振り分ける。 結果は破棄せず保持。
  • 失敗 / 課題が AI 学習の燃料: review / failed を Codegen ループへ再投入。 GPU アイドル時のみ・per-host cooldown・件数上限のゲートで暴走を防ぐ。
  • 壁の構造化 → recipe 化: vision LLM が壁を見て、reasoning モデルがホスト知識に蒸留、それを fetch_recipe(on_after_navigate アクション)に変換して登録。 次回 fetch は recipe で壁を超える。

6. 全体ループ — failed-fetch → 学習 → 次回成功

flowchart LR fetch["mode=fetch ジョブ"] fetch --> outc{"成功?"} outc -->|そのまま成功| C1[completed] outc -->|壁| rev2[review] outc -->|エラー| F1[failed] rev2 --> esc[AI escalation gate] F1 --> esc esc -->|gate 通過| cg["codegen-loop"] cg --> per["perception"] per --> dist["distiller"] dist --> rec["recipe 登録"] rec --> fetch esc -.->|gate ブロック| stop[stop]
学習ループの設計図: review / failed を燃料に AI が試行 → perception で「何故壁にぶつかったか」を構造化 → distiller がホスト知識化 → recipe として次回 fetch を自動補強。 成功判定の精度が足りないと学習が偽信号で汚染されるため、Codegen ループの判定改善が要石になる。
