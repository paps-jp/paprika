Multi-Hub

Edge の nginx が複数 Hub を rr。worker WS だけ sticky。 設定変更は MariaDB write-through + Redis pub-sub で live propagate。 session HTTP はミドルウェアが owner hub へ転送。 デプロイは SoT 一本化(本番 Hub を直接編集しない)。

概要
  • クライアントは rr、worker WS だけ sticky: client は rr で OK(session はミドルウェアが転送、jobs は store-shared)。 worker は sticky で hub_id を安定させ、heartbeat と version gate の往復をシンプルに保つ。
  • 設定変更は live propagate: skills / conv / presets / hosts / settings / engines が MariaDB write-through + Redis pub-sub で全 Hub に伝播。
  • 本番 Hub を直接編集しない: SoT 1 箇所のみ編集し、watcher が hash 分類 → 並列 rsync → /health gate 付き 順次再起動 → worker self-update まで自動化。

3. 設定 cross-hub invalidate(live propagate)

sequenceDiagram participant U as 操作者 participant H1 as hub-A (writer) participant Mar as MariaDB participant R as Redis pub-sub participant H2 as hub-B participant H3 as hub-C U->>H1: PUT /settings /engines /hosts
/skills /conv /presets /extensions /profiles H1->>Mar: write-through H1->>R: PUBLISH paprika:registry:invalidate {type, key} par 並列受信 R-->>H2: event R-->>H3: event end H2->>H2: _apply_event
(MinIO 等 endpoint 変更時は reset_client も) H3->>H3: _apply_event
クライアント cache の罠: MinIO endpoint 変更時に _apply_event が reset_client() を呼ばないと、peer Hub が古い endpoint を握り続けるケースがあった。

4. session 転送ミドルウェア

sequenceDiagram participant C as Client participant N as nginx (rr) participant Y as Hub-Y (landed) participant R as Redis owner index participant X as Hub-X (owner) C->>N: /sessions/{sid}/* N->>Y: rr 転送 Y->>Y: _session_forward_middleware Y->>Y: state.sessions.get(sid)? alt local Y-->>C: handle else remote Y->>R: lookup_owner(sid) R-->>Y: (worker_id, owner_hub) Y->>X: POST /internal/sessions/{sid}/action X-->>Y: result Y-->>C: result end
ミドルウェアに集約しなかった頃は、各エンドポイントごとに転送を呼び忘れる footgun があった。 詳細は Session Lifecycle。

6. デプロイ — SoT 一本化

flowchart TD edit["edit SoT host: /opt/paprika/{server,core}"] edit --> watch["paprika-deploy-watcher (systemd)
30 秒 debounce"] watch --> hash["content-hash 分類

絶対ルール: 本番 Hub やワーカーを直接編集 / scp / 再起動しない。 SoT ホスト 1 箇所のみ編集する。 デプロイ対象は server/ + core/ のみ。 scripts/ や nginx.conf は対象外=手動更新。
