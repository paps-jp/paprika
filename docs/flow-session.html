Session Lifecycle

Session = 持続的なブラウザタブ。OPEN すると lane を専有し、noVNC で human view が可能。idle TTL / 絶対 TTL / DELETE / worker drop の 4 経路で閉じる。Hub 再起動を跨いで生存する(P2)。

概要
  • noVNC 操作だけが keepalive: 「眺めているだけ」は last_active_at を更新しない。クリックしないと既定 300 秒で reap される。
  • cross-hub 転送はミドルウェアで集約: nginx が rr で別 Hub に着地しても、ミドルウェアが Redis owner index を引いてオーナー Hub に proxy する。エンドポイントごとに転送を呼び忘れる footgun を根絶。
  • Hub 再起動を跨ぐ(P2): 完全状態を Redis に永続化、worker は WS drop で破棄せず再 announce 待ち。再起動後の Hub は Redis から rebuild。

keepalive の落とし穴: 受動視聴は last_active_at を更新しない。クリックしないと idle_ttl_s(既定 300 秒)で reap される。長時間張り付くケースは、定期的に keepalive POST を叩くか operator が触る必要がある。
